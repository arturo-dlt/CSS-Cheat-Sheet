In this example, we can have a <div> with a class of .banner and then select the descendant <p> elements. You might find this useful to apply styling to specific HTML elements withing a parent element like a <ul>, <ol>, <section> or <div>.
[href] HTML Attribute You can target a specific attribute in the HTML like the "href" attribute to style everything with a link.
a[href*='Contact'] Specific HTML Attribute You can target something even more granular by adding a specific type and/or attribute values. In the example, this will select every link that contains the word 'Contact'.
a:hover Pseudo-class Selectors These will let you change styles when specific interactions occur, like adding a border in an <input> element to give it more focus, a link <a> changing color when you hover on it or a <button> being "disabled" and then set to "active" after you complete certain tasks. Pseudo-class selectors will give different "states" to elements. Some pseudo-classes are :focus, :visited, :disabled, and :active.
table + p Adjacent Sibling Selector This selects all <p> that directly follow a <table>. Elements that follow one another are called siblings. They're on the same level, or depth. In the HTML markup, elements that have the same indentation are siblings.
table ~ p General Sibling Selector You can select all siblings of an element that follow it. This is like the Adjacent Selector (A + B) except it gets all of the following elements instead of one.
div p:first-child First Child Pseudo-Selector You can select the first child element. A child element is any element that is directly nested in another element. You can combine this pseudo-selector with other selectors. The one used in this example would select all first child <p> elements that are in a <div>.
:only-child Only Child Pseudo-Selector Selects an element that is the only element inside of another one.
:last-child Last Child Pseudo-Selector You can use this selector to select an element that is the last child element inside of another element.
:nth-child(A) Nth Child Pseudo-Selector Selects the nth (Ex: 1st, 3rd, 12th etc.) child element in another element.
:nth-last-child(A) Nth Last Child Pseudo-Selector Selects the children from the bottom of the parent. This is like nth-child, but counting from the back!

CSS TIPS AND TRICKS

Reset the stylesheet

It is considered good practice to RESET the whole default stylesheet as all major web browsers have default stylesheets. These are known as user agent stylesheets. These stylesheets usually have set default values for padding and margin, which is why many developers choose to reset their values so they can start working on a clean slate. User agent stylesheets also usually have the box-sizing set to the default content-box value.

* {

box-sizing: border-box;

margin: 0;

padding: 0;

}

The Box Model

The CSS box model is essentially a box that wraps around every HTML element. It consists of: margins, borders, padding, and the actual content.

Property Values Description
height length, auto, % It defines the height of an element. For example: 500px.
width length, auto, % Just like height, it helps define the dimensions of an element, in this case its width. For example: 90%.
max-height length, auto, % Limits how narrow or wide the element's box can be. This is useful to prevent the content to deform in ways we don't want it to in different screen sizes.
max-width length, auto, % Limits how wide an element's box can be. For example: 1250px.
overflow visible, hidden, scroll, auto Sometimes, when we define a max-height, the content can "spill" outside of its box. This property defines whether to clip the content or to add scrollbars when the content of an element is too big to fit in the specified area.
visibility visible, hidden On the same note, we can define if an element is hidden from our view. The box space it occupies will still be there, but we wouldn't see its content.
border 2px solid green Border properties will let you change the "line" that surrounds an element. You can string the value with a space: width, style and color. You can even specify something with properties that target only one side of the border like border-bottom-width or border-left-style.
border-width thin, medium, thick or px Sets the width of an element's four borders.
border-style none, hidden, dotted, dashed, solid, double, groove, ridge, inset, outset Sets the style of an element's four borders. This one is absolutely necessary for the rest to work.
border-color color Sets the width of an element's four borders.
border-radius lenght, % It lets you create rounded corners. For example: 5px will make it slightly rounded. You can even go as far as creating a perfect circle by first creating an element with the same width and height, and then setting the border-radius equal to half the width of the box, which is 50%.
padding length, % An element's padding is the space between its content and its border. You can target all four sides with a single shorthand value or you can target each side with specific properties like padding-top, padding-right, padding-bottom, padding-left.
margin length, % Margin works exactly like padding. Padding creates extra space within an element, while margin creates extra space around the element. You can also target each side with specific properties: margin-top, margin-right, margin-bottom and margin-left.
margin auto The margin property also lets you center content with the syntax margin: 0 auto. The "0" sets the top and bottom margins to 0 pixels and the "auto" instructs the browser to adjust the left and right margins until the element is centered within its containing element. In order to do so, a width must be set for that element.

CSS TIPS AND TRICKS

Shorthand Values

Some properties, like padding, margin and border can use "shorthand" values. This means they can have from one to four values in one single property. With four values, it will change, for example, the padding in this order: padding-top, padding-right, padding-bottom, padding-left. With two values, the first value would change the padding-top AND padding-bottom, and the second value would change the padding-right AND padding-left. If it has only one value of "10px" then all sides will have a padding of 10px.

This applies to specific properties to change the border as well. For example, we can give different thickness to each side of an element if we add something like border-width: thin medium 5px 10px.

Display and Positioning

Property Values Description
position static This is the default position value in the normal document flow.
position relative The element will be positioned relative to its original static position and it will remain in the normal document flow. To move it, you need to use the top, bottom, right or left properties.
position absolute The element will be positioned relative to its closest positioned parent element. It will be removed from the normal document flow, which means all other elements on the page will ignore it and act like it is not present on the current flow.

Note: For position: absolute to work, it is required to add any type of coordinate to the element, such as top, left, right or bottom properties, even if it has a value of 0px.

Note 2: For position: absolute to work, the parent element HAS to be positioned as well. The default value of position is always unset, so if you don't specify the positioning of the parent, the absolute value of your child element will ignore everything on the page (remove itself from the normal document flow) and just stay in the upper left corner, unrelated to anything (it will base itself on the <body> by default). Absolute positioning will be relative to its CLOSEST eligible/positioned parent element. Say, if you have 4 <div>'s (div 1, div 2, div 3 and div 4) - all nested within the other -, and div 4 has position: absolute, and the only other div with a position set is div 2, then div 4 will position itself relative to div 2, ignoring div 3.
position fixed Lets you fix an element to a specific position on the page (regardless of scrolling), removing it from the normal flow of the document. This technique is often used for navigation bars on a web page.
position sticky Sometimes you will want a particular element to "stick" to a specified position as the page is scrolled further. The sticky value can let you do that. This might be useful when you have an article navigation bar that has to stay on the right hand side of the page while you scroll past a certain point.
top, bottom, left, right length, auto, % These properties are absolutetuly necessary for position to work.
z-index integer This property controls how far back or how far forward an element should appear on the web page when elements overlap (in a stack order like Illustrator layers).

Note: z-index will NOT work unless you add a position to it.
display inline, block, inline-block Every HTML element has a default display value that dictates if it can share horizontal space with other elements.

  • block: These elements fill the entire width of the browser, from left to right, regardless of the size of their content. Some default block elements are headings (<h1> - <h6>), <p>, <div> and <footer>.
  • inline: These elements only take up as much horizontal space as their content requires and can be directly next to other elements. Some default inline elements are <em>, <strong> and <a>.
  • inline-block: These elements keep properties of both, where the elements can appear next to each other and we can specify their dimensions using the width and height properties. Images (<img>) are the best example of default inline-block elements.
float none, left, right We can use the float property to move an element as far right or as far left as it can within the container. HOWEVER, this is not recommended, as moving elements left or right for layout purposes is better suited for tools like CSS grid and flexbox.
clear none, left, right, both The clear property controls the flow next to floated elements (it decides what happens to the element next to a floating element). This is useful because, when we float multiple elements, their different dimensions can affect the layout on the page. Specifically, elements can "bump" into each other and not allow other elements to properly move to the left or right.

CSS TIPS AND TRICKS

Box-Sizing

The default box model has one slight problem: the border-width and padding are not included in the dimensions of the box (width and height), making it difficult to accurately size a box, which will have problems when positioning elements in the future. To avoid this, we can just change the default property for box-sizing from content-box to border-box. This means the width and height properties (and min/max properties) includes content, padding and border. The content area is automatically sized based on the remaining width, after border and padding has been subsctracted.

Text Layout

Property Values Description
letter-spacing length This property sets the spacing between letters. It can sometimes help the readibility of certain fonts or styles. It can take length values in units, such as 2px or 0.5em.
word-spacing length Like letter spacing, you can set the spacing between words. It might help enhance the readibility of bolded or enlarged text and it can also take length values in units.
line-height normal, number, length, % This is basic in text formatting. You usually see this in text editors like Words or Docs. Line height values can be a unitless number, such as 1.2, or a length value, such as 12px, 5% or 2em. Generally, the unitless value is preferred since it is responsive based on the current font size.
text-align left, center, right, justify The text-align property aligns text to its parent element. It can have either one of these values:

  • left: Aligns text to the left side of its parent element, which in this case is the browser.
  • center: Centers text inside of its parent element.
  • right: Aligns text to the right side of its parent element.
  • justify: Spaces out text in order to align with the right and left side of the parent element.

Linking Web Fonts

SOLUTION 1

Web Fonts using <link>

You can find the link to fonts like Google Fonts on their website and then add them to the <head> element in your HTML document. Then you can style it with CSS.

For example, you might have to do this:

<head>

<link href="https://fonts.googleapis.com/css2?family=Roboto:wght@100&display=swap" rel="stylesheet">

</head>

SOLUTION 2

Web Fonts using @font-face

Fonts can also be added using a @font-face ruleset in your CSS stylesheet. Fonts can be downloaded just like any other file on the web.

  • OTF (OpenType Font)
  • TTF (TrueType Font)
  • WOFF (Web Open Font Format)
  • WOFF2 (Web Open Font Format 2)

The different formats are a progression of standards for how fonts will work with different browsers, with WOFF2 being the most progressive. It's a good idea to include TTF, WOFF, and WOFF2 formats with your @font-face rule to ensure compatibility on all browsers.

The ruleset would look something like this:

@font-face {

font-family: 'MyParagraphFont';

src: url('fonts/Roboto.woff2') format('woff2'),

url('fonts/Roboto.woff') format('woff'),

url('fonts/Roboto.ttf') format('truetype');

}

Links and Buttons

For proper web interaction, user agent stylesheets usually style buttons to look "clickable". When we override these styles, we have to take the "interactivity" part of the design into consideration. In the end, you want the final user to easily navigate the site. Your design must feel intuitive.

Property Values Description

It is a shorthand property for text-decoration-line, text-decoration-style, text-decoration-color and text-decoration-thickness. Using an underline an a color:blue is one of the most common ways to style a link on a website. You might use any other CSS properties to style a link, like background-color, font-weight or border.

a:hover any property Using the :hover pseudo-selector you can style the element when a specific "state" is active. In this case, it would activate when you hover the mouse over the element.
cursor pointer, help, wait, grab, not-allowed, zoom-in, etc. This property sets the mouse cursor, if any, to show when the mouse pointer is over an element. Setting it to pointer is a nice way of knowing that something is a link. Usually, browser user agent stylesheets already include these. It is recommended to add it to all the <a> elements, instead of just the a:hover state, as the browser will only change cursor styles on hover by default.

Note: Hover state styling should never be used as the sole indication that something is a link. Users should not be forced to move their mouse over an entire document to tell what might be clickable. Additionally, hover states are not accesible in mobile browsers.
:link any property The :link pseudo-selector is the default state of an unvisited link.
:visited any property The :visited pseudo-selector styles the link related to the user's history with it.
:active any property The :active pseudo-selector is useful for styling links when they have been clicked and are currently active.

Note: The ordering of link pseudo-class rules is important, as one should override the other in this order: :link, :visited, :hover and :active.

CSS TIPS AND TRICKS

Link Styling

Link styles should not be replicated in other page text. It should contrast from the rest of the text color, style or decorations. It keeps the styles consistent and intuitive. Links have to look "clickable". Remember that you cannot rely even on cursor styles because mobile devices don't have the ability to hover any element. Designers should use signifiers to demostrate elements that afford interaction.

Semantics and Context

Also, anchor text, the text itself of a link, should be descriptive of the linked resource. See what I did there?

You can also add additional context with the HTML title attribute, as it is useful for additional context or advisory text for clickable elements. Most browsers will display the text of a title as a tooltip, meaning when a user hovers their cursor over an element, the text will appear in a small box near the cursor. But beware, the title attribute is known to cause a variety of accesibility issues, so it is not always recommended. There are other ways to create tooltips, which include a mixed use of CSS and Javascript, but that is more advanced.
